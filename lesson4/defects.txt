
В классе StringUtils обнаружены следующие баги:

1. В функции def capitilize имеется грамматическая ошибка. 
Верное написание данного слова - capitalize

2. В функции def contains некорректно отрабатывает метод res = string.index(symbol) > -1,
 отображая пустую строку в слове, как True

3. В файле string_utils.py при переносе комменратиев используется \n вмеесте в """. 
При использовании открываюзих и закрывающих """ не нужно использовать \n для переноса.

4 В описании функции def is_empty указан Пример 2: `is_empty(" ") -> True`,
в то время как string = self.trim(string) приводит значение к ("")
